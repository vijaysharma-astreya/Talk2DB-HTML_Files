Intune Data Analysis Pattern

I have identified the intune_assets table, which contains a rich set of data about your Intune-managed devices. Here is a proposed pattern for analysis that we can explore.

Device Inventory Summary

Get a high-level overview of your device landscape. This includes total asset counts, breakdowns by operating system (platform), ownership type (Corporate/Personal), and manufacturer.

Device Compliance Analysis

Understand the compliance posture of your devices. We can visualize the distribution of devices by their compliance state (Compliant, Non-compliant, Unknown) to identify potential security risks.

User-Device Relationship

Map devices to their primary users. This is useful for understanding device allocation and for user-centric reporting and troubleshooting.

Asset Age and Refresh Cycle

Analyze device lifecycle by examining enrollment dates. This can help identify aging assets that may be due for a refresh, helping with budget and hardware planning.

Storage Utilization

Assess the disk space on your devices. We can analyze total vs. free storage to identify devices that are running low on space, which can impact performance and user productivity.

Security Posture

Gain insights into the security settings of your devices. This includes checking the status of disk encryption and detecting if any devices are jailbroken, which are critical security indicators.
